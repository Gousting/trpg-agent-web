# TRPG Agent Web 路线图

Web 端的演进依附于核心引擎的路线图。这里的任务聚焦于"浏览器里看跑团"的体验。

## Phase 1：OBS 直播就绪

**目标：Web UI 能作为 OBS 浏览器源，在直播画布上好看。**

- [ ] **1920×1080 布局适配** — 地图、角色卡、对话三区域在直播画布下不重叠、不溢出。
- [ ] **深色主题** — 跑团直播通常在暗光环境，白底 UI 会刺眼。COC 氛围适配暗色调。
- [ ] **最小化模式** — 隐藏 UI 控件（输入框、设置面板），纯展示模式用于直播采集。
- [ ] **移动端响应式** — 观众在手机上也能看清文字。

## Phase 2：观感增强

- [ ] **骰子 CSS 动画** — 检定时骰子滚动 3D 效果，不用 JS 库，纯 CSS。
- [ ] **场景转场** — 房间切换时地图 0.5s 过渡动画，消除"跳变"。
- [ ] **角色头像** — 调查员/NPC 发言时显示头像（从 ComfyUI 预生成）。
- [ ] **打字机效果** — KP 旁白逐字出现而非整段弹出，增强叙事节奏。
- [ ] **BGM 指示器** — 当前播放的 BGM 名称和氛围标签显示在角落。

## Phase 3：互动

- [ ] **弹幕叠层** — B站弹幕 WebSocket 实时显示在画面上方半透明横滚。
- [ ] **投票 UI** — 观众决策投票的 Web 界面，倒计时 + 实时计数。
- [ ] **多视角切换** — 观众可选择"地图视角""对话视角""角色卡视角"。
- [ ] **回放模式** — 加载历史 session，在浏览器里重播整局游戏。

## 技术改进

- [ ] **WebSocket 替代 SSE** — 支持双向通信，为嘉宾模式做准备。
- [ ] **地图缓存** — 已探索区域缓存为静态 PNG，减少重绘。
- [ ] **性能分析** — 长 session（3h+）下前端内存和渲染性能优化。

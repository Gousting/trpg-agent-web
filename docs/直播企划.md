# AI 跑团直播企划

## 一、项目定位

用本地 AI 主持 COC 跑团并直播，区别于真人 KP 的"手艺活"路线，走**自动化叙事剧场**——AI 主持人驱动多角色自动演绎，观众看故事展开而非看人玩。

核心优势：24/7 可播、模组无限复用、零人力成本、每次演不同剧情线。

## 三、直播产品形态

### 形态A：纯 AI 自动剧场（首发推荐）

KP + 3 个 AI 调查员全自动跑模组，观众纯观看。适合挂机直播、新模组首映。

- 优点：零人工介入，可 24h 轮播
- 缺点：互动性弱，观众容易流失
- 适用场景：深夜档、背景音、模组 demo

### 形态B：弹幕共演（中期目标）

AI 自动跑，但观众通过弹幕投票影响调查员决策。每条弹幕指令经简单过滤后注入 prompt。

- 技术要点：B站弹幕 WebSocket → 关键词提取 → "观众建议：XXX" 注入当前玩家 prompt
- 风险：恶意弹幕破坏叙事（需要黑名单 + LLM 二次过滤）
- 节奏：每 3 轮开放一次投票窗口，避免混乱

### 形态C：嘉宾模式（长期）

1 个真人嘉宾 + 2 个 AI 调查员，AI KP 主持。嘉宾通过 Discord/微信语音接入，STT 转文字后注入。

- 需要：实时语音转文字（faster-whisper 已就绪）
- 适合：特邀场、联动场

## 五、内容策略

### 平台选择

B站为主阵地。理由：COC 跑团在 B站 有成熟受众（搜索"COC跑团"大量视频），弹幕文化天然适配跑团叙事，且 B站 对 AI 内容接受度高于 ACG 纯血圈子。

抖音做切片（高能片段 60s 竖屏），引流到 B站 完整版。

### 模组路线

1. **经典模组汉化跑**（首发）：《鬼屋》《黑暗边缘》等短模组，1-2 小时适合直播
2. **原创模组**（中期）：根据弹幕反馈迭代自己写的模组
3. **观众投稿模组**（长期）：观众提交创意，AI 即兴跑

### 排播节奏

- 每周 2-3 场正式直播（晚 8-10 点黄金档）
- 其余时间挂机轮播过往精彩场次
- 切片投 B站 动态 + 抖音

## 六、风险与应对

| 风险 | 应对 |
|------|------|
| AI 叙事崩坏（逻辑断裂/重复） | 加 sanity check，3 轮无进展自动切场景或触发事件推动 |
| 弹幕恶意指令破坏游戏 | 黑名单 + LLM 过滤 + 仅取前 N 条高频建议 |
| ACG 圈子反 AI 情绪 | 标题避免"AI"，用"自动剧场""程序叙事"，不立"替代真人 KP"人设 |
| Ollama 模型掉线 | 加健康检查 + 自动回退到硅基流动 API |
| 内容同质化 | 多模组轮换 + 随机种子保证每次剧情不同 |
